## 学习内容总结

### 1. 完整训练循环 (01_complete_training_loop.py)

- 理解了 **epoch**（完整遍历一次全部数据）和 **batch**（每次只取一小批）的区别
- 掌握了双层循环结构：外层 `for epoch` + 内层 `for batch_x, batch_y in loader`
- 明确了每个 batch 必须执行的五步：
  1. `optimizer.zero_grad()` — 清零梯度（防止梯度累积）
  2. `y_pred = model(batch_x)` — 前向传播
  3. `loss = criterion(y_pred, batch_y)` — 计算损失
  4. `loss.backward()` — 反向传播
  5. `optimizer.step()` — 更新参数
- 用 `epoch_loss / len(loader)` 计算 epoch 平均 loss（不能只看最后一个 batch）
- 验证了模型能从 20 个固定样本学到真实权重 [2, 3, -1] 和偏置 1

### 2. Dataset 与 DataLoader (02_dataloader.py)

- 对比了手动切批（繁琐）vs DataLoader（简洁）的代码量差异
- 掌握了 **TensorDataset**：直接封装内存中的张量，适合快速原型
- 掌握了 **自定义 Dataset**：继承 `torch.utils.data.Dataset`，必须实现：
  - `__len__`：返回样本总数（DataLoader 计算 batch 数量用）
  - `__getitem__`：按索引返回单个样本
- 理解了三个关键参数：
  - `batch_size`：每批样本数
  - `shuffle=True`：训练集打乱，验证/测试集不打乱
  - `drop_last=True`：丢弃最后不足一批的样本

### 3. 训练集/验证集拆分 (03_train_eval_split.py)

- 用 `random_split(dataset, [train_size, val_size])` 实现 80/20 拆分
- 掌握了 `model.train()` / `model.eval()` 的正确切换时机
- 学会了 `with torch.no_grad():` 在验证时关闭梯度记录（节省内存 + 加速）
- 实现了每个 epoch 同时输出 `train_loss` 和 `val_loss`

## 核心概念理解

### epoch vs batch 的关系

```
全部数据（20 个样本）
    ↓  DataLoader(batch_size=5)
每个 epoch 包含 4 个 batch
    ↓  外层循环 × num_epochs
模型参数更新 4 × num_epochs 次
```

### 为什么 zero_grad() 必须在每个 batch 前调用

PyTorch 默认**累积**梯度。不清零会把上一个 batch 的梯度叠加进来：

```
batch 1 梯度: [0.5, -0.3]
batch 2 正确: [0.8,  0.2]
batch 2 错误: [1.3, -0.1]  ← 累积了 batch 1 的残留
```

### model.train() vs model.eval()

| 状态 | Dropout | BatchNorm | 何时调用 |
|------|---------|-----------|---------|
| train() | 随机丢弃 | 用当前批统计量 | 训练循环开始前 |
| eval() | 关闭 | 用全局统计量 | 验证/推理前 |

本模型无 Dropout/BN，但养成习惯在复杂模型中至关重要。

### torch.no_grad() 的作用

验证时不需要反向传播，不记录计算图可以：
- 节省约 50% 显存/内存
- 加速前向计算

## Core Concept Understanding

### Relationship Between epoch and batch

```
All data (20 samples)
    ↓  DataLoader(batch_size=5)
Each epoch contains 4 batches
    ↓  outer loop × num_epochs
Model parameters updated 4 × num_epochs times
```

### Why zero_grad() Must Be Called Before Every Batch

PyTorch **accumulates** gradients by default. Without zeroing, the previous batch's gradients stack onto the current ones:

```
Batch 1 grad:    [0.5, -0.3]
Batch 2 correct: [0.8,  0.2]
Batch 2 wrong:   [1.3, -0.1]  ← batch 1 residue accumulated
```

### model.train() vs model.eval()

| State | Dropout | BatchNorm | When to call |
|-------|---------|-----------|--------------|
| train() | randomly drops | uses current batch stats | before training loop |
| eval() | disabled | uses global stats | before validation/inference |

This model has no Dropout/BN, but the habit is critical for complex models.

### What torch.no_grad() Does

Validation doesn't need backpropagation. Not recording the computation graph:
- Saves ~50% GPU/CPU memory
- Speeds up forward computation
